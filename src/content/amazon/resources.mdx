## Appendix A: AWS Project Ideas
Pick one, finish it, and document it well.

* **Campus FAQ Chatbot (RAG):** Create a Knowledge Base with a handful of campus PDFs and a website crawl; build a small web UI. ([Workshop](https://workshops.aws/card/bedrock), [Blog](https://aws.amazon.com/blogs/machine-learning/build-a-contextual-chatbot-application-using-knowledge-bases-for-amazon-bedrock/))
* **Long‚Äënote Summarizer:** Summarize lecture notes with citations back to source paragraphs. Try multiple models via Bedrock to compare styles.
* **Syllabus Q&A Assistant:** Answer questions about due dates and policies by grounding to a course syllabus; include a ‚ÄúShow sources‚Äù toggle.
* **Support QnABot:** Start from the [AWS Solution and swap in Bedrock for answers](https://aws.amazon.com/solutions/implementations/qnabot-on-aws/).  
* **Agent that runs simple tasks:** Use an Agent for Bedrock to call a weather or calendar API and return structured JSON back to your app.
* **Code‚Äëgeneration assistant:** Experiment with Bedrock models capable of code generation. Build an agent that generates boilerplate code, then use guardrails to filter unsafe output.
* **Guardrail experimentation:** Set up a guardrail using Amazon Bedrock Guardrails and test the different policies. Create content‚Äëfilter and prompt‚Äëattack policies, define denied topics and word filters, and test with sample prompts.

Each of these projects demonstrates different aspects of the AWS AI/ML ecosystem. They're excellent for learning because they expose you to various AWS services working together. As always, focus on understanding the architecture and making thoughtful service choices.

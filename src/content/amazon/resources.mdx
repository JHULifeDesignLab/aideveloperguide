## Appendix A · Sample project ideas

* **Text summarizer** – Use Amazon Bedrock foundation models (e.g., Anthropic Claude or Amazon Titan) with LangChain to summarise long documents.  The *architecture patterns* module demonstrates text summarization with Bedrock models and LangChain:contentReference[oaicite:40]{index=40}.  Build a serverless API (e.g., AWS Lambda) that takes a PDF or web page and returns a concise summary.
* **Question‑answering with RAG** – Implement a retrieval‑augmented generative (RAG) system by creating a knowledge base of your course notes.  The Bedrock application‑components module shows how to work with datasets, embeddings and RAG:contentReference[oaicite:41]{index=41}, while the architecture patterns module includes a question‑answering example:contentReference[oaicite:42]{index=42}.  Use LangChain’s retrievers and Bedrock’s Knowledge Bases to answer questions about the material.
* **Chatbot with memory and agents** – Combine Bedrock with LangChain to create a conversational interface that can remember context and call external tools.  The LangChain module teaches how to construct prompts, sequence chains, manage memory and use agents:contentReference[oaicite:43]{index=43}.  Extend the example chat‑bot demonstration with additional features like persona prompts or integration with Slack.
* **Code‑generation assistant** – Experiment with Bedrock models capable of code generation.  The architecture patterns module lists code‑generation examples and shows how to use Bedrock models for programming tasks:contentReference[oaicite:44]{index=44}.  Build an agent that generates boilerplate code, then use guardrails to filter unsafe output.
* **Guardrail experimentation** – Set up a guardrail using Amazon Bedrock Guardrails and test the different policies.  Follow the steps described in the AWS “AI safety controls at scale” article: create a guardrail, configure content‑filter and prompt‑attack policies, define denied topics and word filters, and test with sample prompts:contentReference[oaicite:45]{index=45}.  Document the impact on the model’s responses and share your findings.
